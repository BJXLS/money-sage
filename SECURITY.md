## 报告漏洞

我们非常重视 MoneySage 的安全性。如果你发现了安全漏洞，请**不要**通过公开的 Issue 报告。

### 如何报告

请通过以下方式之一报告安全漏洞：

1. **私密安全咨询**（推荐）

   - 前往仓库的 Security 标签
   - 点击 "Report a vulnerability"
   - 填写详细信息

2. **电子邮件**
   - 发送邮件至: [your-security-email@example.com]
   - 主题：[SECURITY] MoneySage Security Issue

### 报告内容应包括

- 漏洞的详细描述
- 复现步骤
- 受影响的版本
- 潜在的影响
- 可能的修复建议（如果有）

### 响应时间

- **初次回复**: 48 小时内
- **漏洞确认**: 7 天内
- **修复发布**: 根据严重程度，14-30 天

### 安全更新流程

1. **验证**: 我们会验证报告的漏洞
2. **修复**: 开发修复补丁
3. **测试**: 全面测试修复方案
4. **发布**: 发布安全更新版本
5. **公告**: 在 CHANGELOG 中说明（不泄露细节直到大多数用户更新）

## 安全最佳实践

### 用户

- **定期更新**: 始终使用最新版本的 MoneySage
- **下载来源**: 仅从官方 GitHub Releases 下载
- **备份数据**: 定期导出和备份你的财务数据
- **操作系统**: 保持 Windows 系统和 WebView2 更新
- **防火墙**: 如果使用 AI 功能，确保网络连接安全

### 开发者

如果你为 MoneySage 贡献代码，请遵循以下安全实践：

- **依赖审查**: 审查所有新依赖的安全性
- **代码审查**: 所有 PR 都需要代码审查
- **输入验证**: 验证和清理所有用户输入
- **敏感数据**: 不要在代码中硬编码密钥或凭证
- **错误处理**: 避免错误消息泄露敏感信息

## 已知的安全考虑

### 数据存储

- **本地数据库**: 数据存储在本地 SQLite 数据库中
- **未加密**: 当前版本数据库未加密（计划在未来版本添加）
- **文件权限**: 依赖操作系统的文件权限保护
- **建议**:
  - 不要在共享电脑上使用
  - 使用磁盘加密（如 BitLocker）
  - 定期备份数据

### AI 功能

- **API 密钥**: LLM API 密钥存储在本地配置中
- **数据传输**: 使用 AI 功能时，记账描述会发送到 LLM 服务
- **隐私**: 不要在描述中包含敏感个人信息
- **建议**:
  - 使用自己的 API 密钥
  - 了解所使用 LLM 服务的隐私政策
  - 考虑使用本地 LLM（如支持）

### 网络安全

- **HTTPS**: 所有外部 API 调用使用 HTTPS
- **证书验证**: 启用 SSL 证书验证
- **超时**: 合理的网络请求超时设置
